# Waterfall Chart

**Use when**: decomposing the net change between two values into contributing increases and decreases (P&L, HR headcount changes, subscription churns and adds).  
**Avoid when**: scale disparity is extreme — a 10,000-person base with 5-person changes produces invisible floating bars; gains and losses are randomly interspersed without a logical grouping; the audience is unfamiliar with floating bars.

## Key rules

- Group all gains together, then all losses — avoids a confusing zig-zag staircase
- Use distinct colors for positive vs negative bars (e.g., green/blue for gains, red/orange for losses)
- Draw horizontal guide lines connecting bar edges to help readers track the cumulative total
- Label each bar's value directly — readers can't judge floating bar lengths without labels
- For extreme scale disparities, show only the delta bars and note the starting value as text

## Anti-patterns

- Random ordering of gains and losses — creates visual chaos and makes the cumulative story unreadable
- No connecting guide lines — readers lose track of where the running total is
- Truncated y-axis — same problem as with bar charts; distorts perceived magnitude
