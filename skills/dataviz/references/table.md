# Table

**Use when**: data has mixed units or types that resist a single visual encoding; the audience needs to look up or compare specific values; supporting detailed data in an appendix alongside summary charts.  
**Avoid when**: presenting live — an audience cannot listen and read simultaneously; showing trends or patterns (a chart communicates these faster and more clearly).

## Key rules

- Align text left, numbers right (decimal-aligned when precision matters)
- Minimize borders and shading — use whitespace to separate rows/columns instead of grid lines
- Place the most important metric in the upper-left; readers scan top-left to bottom-right
- Order rows by the primary comparison dimension (value, date, priority) — not alphabetically unless lookup is the purpose
- Add visual encoding where useful: heatmap color, sparklines, or data bars to surface patterns without replacing numbers
- Use super-category labels (grouped headers) to reduce repetition in the left column

## Anti-patterns

- Tables as the primary display during a live presentation — the audience tunes out to read
- Showing every column from the raw dataset — select only what answers the questions from Phase 1
- All-borders "spreadsheet" look — heavy gridlines increase visual noise; use minimal borders or none
