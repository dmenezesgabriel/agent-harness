# Slopegraph

**Use when**: comparing values at exactly **two** time points or conditions; the story is which series rose, fell, or stayed flat — and by how much.  
**Avoid when**: middle time points carry essential context (use a line graph); categories are unrelated (lines become meaningless spaghetti); absolute values are the focus rather than change.

## Key rules

- Zero baseline not required — axes can be scaled to the data range to maximize slope visibility
- Label both endpoints with category name and value
- Slope steepness communicates rate of change — don't compress the y-axis to the point where all slopes look flat
- For 3+ time points, switch to a line graph
- Use color or line weight to highlight the 1–2 series that carry the story

## Anti-patterns

- Slopegraph with 3+ columns — the middle values need a line graph, not a slope
- Unrelated categories — readers try to connect slopes that have no meaningful relationship
- All slopes nearly flat with identical values — use a bar chart for the comparison instead
