Write a PRD for this product brief.

Team leads at project-based agencies currently open each active project one by one before stand-up to understand status, blockers, and overdue work. This takes 10-15 minutes every morning and causes missed blockers when a project is skipped.

The first release should help team leads prepare for stand-up faster. Developers should not receive extra notifications from this release. Existing project permissions must still apply. Success means reducing time-to-status-summary during stand-up preparation from 12 minutes to under 3 minutes for pilot teams.

Do not design the architecture or implementation plan.
