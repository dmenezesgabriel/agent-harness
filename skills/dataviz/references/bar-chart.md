# Bar Chart

**Use when**: comparing quantities across categories; ranking items on a single metric.  
**Avoid when**: showing part-to-whole relationships; differences are so small that a truncated axis would be tempting.

## Key rules

- Zero baseline is **mandatory** — the full bar range must be visible or relative comparisons are wrong
- Sort bars by value (largest → smallest or reverse) unless a natural order exists (e.g., months, age groups)
- Gap width ≈ 30–40% of bar width — thinner than the bars, wider than nothing
- Label values inside bars to prevent clutter; use a legend beneath the title only when internal labels aren't feasible
- Use horizontal bars when category names are long

## Anti-patterns

- Truncated y-axis to exaggerate small differences
- Clustered bars with too many series — switch to small multiples or separate charts past 3–4 series
- Alphabetical sorting when value-based sorting would reveal the story faster
